The purpose of these user guidelines is to outline the functionality and use of the Procurement Enquiry Database.

This database is a system for recording the key details from each procurement, and attaching scanned copies of the full audit trail, as a means of quickly being able to access historical procurement documentation for future reference - particularly during internal and external/donor audits.

Search Function



There is one more icon/button to discuss, and that is the Search Options button. This will open the main search options (the same as using "Ctrl+F") and can be used to search in a number of different ways. Hitting this button will open the following search options window:



Things To Consider


When searching there are two options/functions to consider, depending on what you are searching for. There are two drop-down options called Look In: and Match:



Look In:

Under this heading there are the following two options:

Current Field
This means that it will only search in the current field. This is useful for when you're looking for something specific. An example would be an SR No. Rather than slowing down the search by searching the whole database, simply select the relevant field (in this case the SR field) before hitting the search button.

Current Document
This means useful for when you may not be sure what field the data is contained in (e.g. perhaps the GRN number, or a piece of text), as this will search every field and record in the database.

Match:

Under this heading there are the following three options:

Whole Field
This will search for an EXACT match of what you have entered, meaning that no other data can be contained in the field.

Any Part Of The Field
This is the broadest type of search as it will search for data at the start, end and middle of the sentence/field.

Start Of Field
This will search at the start of the string to check for the data entered.

Setting Up A New Enquiry

Follow The Steps Below

1. How To Open A New Enquiry

At the top of the Enquiry Coverpage you will see a number of icons (which are explained in detail later on). When you hover over these buttons, text will appear to tell you what each button does.

To Add A New Enquiry click the one on the left (icon of a document with a "+" sign).



The database is designed so that the "Enquiry No:" field will automatically incremement each time a new enquiry/record is created. In the screenshot above, the Enquiry No is 6931 - if we created a New Enquiry now, the number would be 6932, and so on.

2. Add New Enquiry (minimum info)

The minimum data that should be entered when setting up a new enquiry is the following:

- Enquiry No (automated)
- Country (dropdown list)
- Responsibility (dropdown list)

Responsibilty is a mandatory field

The following data should be entered if known at this stage:

- SR No (unless is it "Quote Only" at this stage)
- Date Received
- Donor (can type more than one)
- Description
- Status (default "No Action Taken")



For the Description field, this is only meant to be a general and brief description of the items. For example, if buying pharmaceuticals, then rather than listing each drug, "Pharmaceuticals" would suffice. (The same way the above example does not list all of the consumables). Comments can also be added at this stage if desired.

In Microsoft Access there is no need to Save your progress. Data and records are automatically saved when you move onto the next record / close the database.

3. Set Up Corresponding Folder on Shared Drive

Now that we have entered the minimum data and have our New Eqnuiry set up, it is vitally important to set up a New Enquiry Folder on the shared drive. This is where we will later archive our scanned documents and electronic paper trail. The location on the server is as follows:

\\10.1.0.172\SupplyChain\Enquiries\
CountryFolders (this not a link, but you can copy and paste)

To Open This Folder from within the database click this icon (icon of a document with a "+" sign).

If a warning message appears, simply click "Yes"

The following window should then open:



You will see that there a folder for each country. If one does not exist for the country in question, then simply create a new one. Within each folder there will be a corresponding folder for each existing enquiry (Starting with the pre-fix "Enq" and followed by the enquiry number. Set up a new folder for the enquiry that you just created.

Once the folder is created, click in the top toolbar and copy the filepath.

Now return to the Enquiry Database and paste this filepath into the field labelled Audit Trial - File Link.

This section includes the general info for the requistion and contains the minimum data required when setting up a new enquiry. However, it is important to note the following two ares:

- Complete (to be ticked when Enquiry is complete/closed. All incomplete enquiries will appear on the SR Status report)

- Audit Trail - File Link: Paste the filepath for the location of the corresponding Enq Folder on the shared drive.

Procurement Details



This section includes info pertaining to the purchase of the items (supplier details, PO and Invoice details, etc.), as well as a link to the Audit Trail folder. It is important to note the following checkboxes:

- Marine Insurance (to be ticked to be included in the marine insurance data export. (explained below)

- Eligible For VAT Reclaim (to be ticked to be included in the VAT Reclaim data export).

- Sent For Payment (to be ticked to indicate that the goods have been receipted in Micorsoft Great Plains)

Marine Insurance Details



This section includes info pertaining to marine insurance details. To be sure that the item will be included in the Marince Insurance Export the box on the far-left should be ticked.(see Reports section below on how to run the export)

Next the value should be entered, followed by the Origin (which can be obtained from the shipping docs) and lastly the Mode of Transport (dropdown menu).

Remember

For the Insured Value the user can simply copy and paste the PO Value, but this deliberately does not defaul to this value, as there will be times when the Insured Value differs from the purchase value (e.g. For IT Shipments we generally only place a PO for freight costs, where the insured value should be that of the goods)



Only items shipped that have NOT been insured by the supplier/freight forwarder should be included in the Marine Insurance.
