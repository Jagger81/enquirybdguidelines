The purpose of these user guidelines is to outline the functionality and use of the Procurement Enquiry Database.

This database is a system for recording the key details from each procurement, and attaching scanned copies of the full audit trail, as a means of quickly being able to access historical procurement documentation for future reference - particularly during internal and external/donor audits.

Understanding The Layout

Follow The Steps Below

1. Opening the Enquiry Database and Enable Content

The Enquiry Database is located on the shared drive, at the following location:

\\10.1.0.172\SupplyChain\Enquiries (this not a link, but you can copy and paste)

Enable Content


When you open the database you might be presented with a Security Warning message - if so, simply hit the Enable Content button. This may have to be performed every time you open the Enquiries Database.

2. Main Menu

Main Menu Screenshot

Enquiry Options, Supplier Options, Search Options and Reporting Options

For now, the only one that is relevant (as highlighted above) is Enquiry Options. The other 3 tabs are under development and can be expanded on if/when the need arises.

The previous version of the database allowed for up to 5 "Activities" (i.e procurement items) to be entered under the same enquiry - this has since been decreased to 3, to allow for additional functionaility. If the user wishes to view older Enquiries in that format they can hit the button at the bottom right of the menu.

In the top right hand corner there is a link to the Intranet.

Setting Up A New Enquiry

Follow The Steps Below

1. How To Open A New Enquiry

At the top of the Enquiry Coverpage you will see a number of icons (which are explained in detail later on). When you hover over these buttons, text will appear to tell you what each button does.

To Add A New Enquiry click the one on the left (icon of a document with a "+" sign).



The database is designed so that the "Enquiry No:" field will automatically incremement each time a new enquiry/record is created. In the screenshot above, the Enquiry No is 6931 - if we created a New Enquiry now, the number would be 6932, and so on.

2. Add New Enquiry (minimum info)

The minimum data that should be entered when setting up a new enquiry is the following:

- Enquiry No (automated)
- Country (dropdown list)
- Responsibility (dropdown list)

Responsibilty is a mandatory field

The following data should be entered if known at this stage:

- SR No (unless is it "Quote Only" at this stage)
- Date Received
- Donor (can type more than one)
- Description
- Status (default "No Action Taken")



For the Description field, this is only meant to be a general and brief description of the items. For example, if buying pharmaceuticals, then rather than listing each drug, "Pharmaceuticals" would suffice. (The same way the above example does not list all of the consumables). Comments can also be added at this stage if desired.

In Microsoft Access there is no need to Save your progress. Data and records are automatically saved when you move onto the next record / close the database.

3. Set Up Corresponding Folder on Shared Drive

Now that we have entered the minimum data and have our New Eqnuiry set up, it is vitally important to set up a New Enquiry Folder on the shared drive. This is where we will later archive our scanned documents and electronic paper trail. The location on the server is as follows:

\\10.1.0.172\SupplyChain\Enquiries/Country Folders (this not a link, but you can copy and paste)

To Open This Folder from within the database click this icon (icon of a document with a "+" sign).

If a warning message appears, simply click "Yes"

The following window should then open:



You will see that there a folder for each country. If one does not exist for the country in question, then simply create a new one. Within each folder there will be a corresponding folder for each existing enquiry (Starting with the pre-fix "Enq" and followed by the enquiry number. Set up a new folder for the enquiry that you just created.

Once the folder is created, click in the top toolbar and copy the filepath.

Now return to the Enquiry Database and paste this filepath into the field labelled Audit Trial - File Link.

4. Print Coverpage

Once the minimum data has been entered and a corresponding folder has been set up in the Country Folders on the shared drive, you can now print off the coverpage. It is extremely important that you do not use the general File > Print option (or "crtl+p") as this may inadvertently print ALL of the coverpages in the database, for each record, resulting in hundreds of pages being printed.
